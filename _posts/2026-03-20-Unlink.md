# Unlink

unlink是一个把空闲chunk从双向链表拿出来的过程，基本就是数据结构中双向链表的删除。在ptmalloc 2中Unsorted bin、Small bins、Large bins都采用双向链表结构。unlink操作会在malloc或者free时触发，malloc很好理解，分配一个空闲块自然会从链表中取一个空闲块下来；而free的话，虽然是放一个空闲块进列表，但是由于存在相邻空闲块的合并操作，放之前会把相邻空闲块从链表中取出，然后合并了再放回列表，因此也有unlink操作。后者的利用较为容易。

## 原理&利用

首先先讲一下unlink的操作，分为以下几步

```
FD = P->fd
BK = P->bk

FD->bk = BK
BK->fd = FD
```

第一步，分别取当前chunk指针P的fd位和bk位，找到P的前一个chunk和后一个chunk；

第二步，令前一块FD的bk指针指向P的后一块，再令后一块BK的指针fd指向P的前一块，这样P的前后两块就能互相连接了。

在glibc 2.3.4（不是2.34哦）之前，这个过程可以被利用来往任意地址写东西，利用如下：

```
32 bit system
FD = P->fd, and P->fd = target_addr - 12
BK = P->bk, and P->bk = expect_value

FD->bk = BK
BK->fd = FD
```

首先，通过一些漏洞，把P的fd和bk覆盖了，这样unlink的时候，FD就会被设置为target_addr - 12，BK则是expect_value；

重点在FD->bk = BK，因为FD->bk实际上是取FD的指针的地址在加上bk的偏移，在32位中偏移就是12，而FD是等于target_addr - 12的，一加一减抵消了，赋值符号左边其实就是target_addr，而BK是expect_value，所以整个FD->bk = BK的语句，实际上执行的是target_addr = expect_value ，就形成了任意地址写。不过注意BK->fd = FD，其实也会往*(expect_value + 8)的地方写target_addr，必须要确保这个expect_value + 8的位置也是可写的，否则会unlink不了。

简单的原理弄完后来看复杂的，在2.3.4后，有了一条检查：

```C
if (__builtin_expect (FD->bk != P || BK->fd != P, 0))                     
  malloc_printerr (check_action, "corrupted double-linked list", P, AV);  
```

会检查FD当前的后一个和BK的前一个指针指向的到底是不是P，漏洞虽然还能用，但是只能写特定地址了：

```
64 bit system
FD = P->fd, and P->fd = P - 0x18
BK = P->bk, and P->bk = P - 0x10

FD->bk = BK
BK->fd = FD
```

将fd和bk覆盖为上述值之后，能通过检查，因为FD  =  P - 0x18，所以FD->bk = P-0x18 + 0x18 = P，而BK->fd = P - 0x10 + 0x10 = P，满足前后都是P的约束。最终赋值，FD->bk = BK，使P = P - 0x10，而下一条会接着覆盖，使P = P - 0x18。基本上，现在的unlink利用就只能使一个块指针，指向它原本的位置-0x18了。
