RC4是一种流加密算法，密钥长度可变，属于对称加密算法。所谓流密码，即用长度和明文相等的密钥流，去对明文进行异或来加密与解密，其思想来源于绝对安全的一次一密OTP--密钥长度等于明文长度，且真随机，用完即弃，保证每次加密的密钥完全不同。

#RC4原理
## 加密步骤

整个加密过程可以分为三步，第一步，初始化S盒；第二步，生成密钥流；第三步，用密钥流加密。

### 初始化S盒（KSA过程）

S盒是Substitution-box，替换盒的意思，用于实现密码学中的混淆，不同密码技术的S盒形态也不同。在RC4中，初始化S盒的代码如下。

```c
// 1. 初始化：按顺序填充S盒（此时S盒是完全有序的）
for i = 0 to 255:
    S[i] = i;

// 2. 密钥调度算法（KSA）：利用用户密钥将S盒彻底打乱
j = 0;  // 临时索引，用于增加混乱度
for i = 0 to 255:
    // 关键点：循环利用密钥中的字节（i % keylen 保证不越界）
    j = (j + S[i] + key[i % keylen]) % 256;
    
    // 交换 S[i] 和 S[j]，完成一次洗牌
    swap(S[i], S[j]);
```

S盒的初始化靠的是用户输入的密钥，可以说，用户自定义的密钥决定了S盒，利用密钥去使S盒无序化的操作也被称为密钥调度算法（KSA）。

### 生成密钥流（PRGA过程）

生成密钥流则主要依赖于刚刚初始化的S盒，每生成一个字节的同时也对S盒做替换操作，保证其混乱性，其生成算法如下：

```c
// 全局或外部传入的S盒，以及状态索引 i, j（需持续保留）
// PRGA（伪随机生成算法）
S[256];
i = 0, j = 0;   // PRGA 的初始状态必须为 0

// 功能：按字节生成伪随机密钥流
// 每次调用，S盒内部状态都会发生一次交换变化
while Generating:
    // 1. 更新索引
    i = (i + 1) % 256;
    j = (j + S[i]) % 256;

    // 2. 交换 S[i] 和 S[j]（让S盒持续自我搅动）
    temp = S[i];
    S[i] = S[j];
    S[j] = temp;

    // 3. 根据交换后的位置取出密钥流字节
    int t = (S[i] + S[j]) % 256;
    keystream_byte = S[t];

```

从S盒生成密钥流的操作，也被成为伪随机生成算法（PRGA）。

### 加密

加密直接用刚刚得到的密钥流去逐字节异或明文，即可得到密文，加密过程通常和生成密钥流过程写在一起，算法如下：

```c
// 功能：对数据进行加解密（原地操作）
// 输入 data：指向待处理数据的指针（加密时是明文，解密时是密文）
// 输入 length：数据的字节长度

// 逐字节处理明文
for (int n = 0; n < length; n++):
    data[n] = data[n] ^ keystream_byte;

```

## 解密步骤

解密过程不关心各种初始化，异或又是个可逆运算，用密钥直接异或密文即可得到明文。

## 完整实现

用C代码，完整实现上述过程：

```c
#include <stdio.h>
#include <stdint.h>

void rc4_init(uint8_t *s, uint8_t *key, uint32_t len) {
    int i;
    int j = 0;
    char k[256];
    uint8_t tmp = 0;

    for(i = 0; i < 256; i++) {
        s[i] = i;
        k[i] = key[i % len];
    }

    for (i = 0; i < 256; i++) {
        j = (j + s[i] + k[i]) % 256;
        tmp = s[i];
        s[i] = s[j];
        s[j] = tmp;
    }
}

void rc4_crypt(uint8_t *s, uint8_t *buf, uint32_t len)
{
    int i = 0, j = 0, t = 0;
    uint32_t k = 0;
    uint8_t tmp;

    for(k = 0; k < len; k++) {
        i = (i + 1) % 256;
        j = (j + s[i]) % 256;

        tmp = s[i];
        s[i] = s[j];
        s[j] = tmp;

        t = (s[i] + s[j]) % 256;
        buf[k] ^= s[t];
    }
}

int main() {
    uint8_t S[256];
    uint8_t key[] = "test_key";
    uint8_t data[] = "Hello World!";

    rc4_init(S, key, sizeof(key) - 1);
    rc4_crypt(S, data, sizeof(data) - 1);

    for (int i = 0; i < sizeof(data) - 1; i++)
        printf("%02X ", data[i]);
    printf("\n");

    return 0;
}
```



# RC4逆向

逆向主要依据两个特征，其一是S盒的初始化，其二是按明文长度生成密钥流，来看一道题目。

可以看到想要输入flag正确，首先得确保flag是32位的（还有一位是换行），然后经过了sub_401160()、sub_401000()、sub_4010e0()三个函数的处理，最后比较内存中两个位置的值是否相等。

## 特征分析

先看看sub_401160，可以先别管前面又是移位，又是各种操作的是干嘛，可以直接看到最后用了个if else，if满足时当前值变成"="，不满足时则是从一个数组里面取值

![ref2](/assets/images/2026-07-06-RC4-Reverse/ref2.png)

跟踪数组，这不就是base64编码表吗，基本可以确定这个函数是在做base64编码操作了。这种一长串代码各种v1、v2、v3还是不好看的，找特征就可以了，然后可以用动态调试来验证，自己从头把所有代码看到尾不现实。

![ref3](/assets/images/2026-07-06-RC4-Reverse/ref3.png)

下一个，sub_401000()，S盒的初始化特征也是很明显的，因为RC4不指定密钥长度，而S盒的长度又固定，为了使任意密钥生成S盒就必须有模`密钥长度`的操作，同理为了在初始化S盒时的j值不数组越界会有模`256`的操作，为了用有限的S盒生成任意长度的密钥流也必须要有模`256`的操作。放在这里`v7 = v4++ % a3;`做的就是S盒初始化的时候的模密钥长度操作，`v3 = (*result + (char)result[v8] + v3) % 256;`做的就是S盒内部交换时，取不越界的j并用S[j]与S[i]交换的操作，这里的`*result++ = a1[v3];a1[v3] = v10;`也对应交换的操作，因此sub_401000()基本可以确认是在做RC4的S盒初始化操作。

![ref4](/assets/images/2026-07-06-RC4-Reverse/ref4.png)

最后一个函数sub_4010e0()，看到最后的异或，以及前面的模256，结合前一个函数，就已经可以确认是在做RC4的生成密钥流同时加密的操作了。

![ref5](/assets/images/2026-07-06-RC4-Reverse/ref5.png)

## 具体解题

由上可知，大概有两种不同的解题方法，一种是手动还原，手工找到RC4初始化S盒的密钥，然后自己模拟RC4初始化生成密钥流，再用密钥流去解密，如下：

```python
import base64
from Crypto.Cipher import ARC4

target = b"\x20\xC3\x1A\xAE\x97\x3C\x7A\x41\xDE\xF6\x78\x15\xCB\x4B\x4C\xDC\x26\x55\x8B\x55\xE5\xE9\x55\x75\x40\x3D\x82\x13\xA5\x60\x13\x3B\xF5\xD8\x19\x0E\x47\xCF\x5F\x5E\xDE\x9D\x14\xBD"
key = b"flag{this_is_not_the_flag_hahaha}"
rc4 = ARC4.new(key)
tmp = rc4.decrypt(target).decode()

old_table = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789+/="
new_table = "ABCDEFGHIJSTUVWKLMNOPQRXYZabcdqrstuvwxefghijklmnopyz0123456789+/="

general = ''
for i in tmp:
	general += old_table[new_table.find(i)]
print(base64.b64decode(general).decode())
```

注意到生成RC4密钥流时，使用了python的标准库去生成了一个RC4对象，但是题目不一定总是能使用标准RC4的，就像base编码魔改编码表，RC4也能魔改S盒位数、KSA时交换的位置、PRGA生成密钥流的具体步骤等等，让我们无法使用标准实现去解题，这时候还要还原的话就只能具体去分析它魔改的地方，不过IDA本身就不能完全还原源代码，各种变量的分析还是挺麻烦的，如果上一点混淆了就更麻烦了。

因此，第二种方法是动态调试，特别是如果程序没有检测调试状态的话，一定要去试试，考虑到RC4的本质只是异或，如果能找到异或用的密钥流，配合密文就直接游戏结束了，实战中在密钥流的生成处下一个断点，就可以从内存中逐步读取出密钥流，这样的方式就比慢慢还原算法简单的多。

在解题时，最首要的事情还是先依据特征去判断是什么算法，又或者根据比较明显的调用去判断做什么操作，然后再根据具体情况具体分析，而不是上来就直接硬啃代码，基本上所有逆向都是这个思路。
