# MD5原理
## 步骤

整个过程可以分为三步，第一步，初始化缓冲区；第二步，消息填充；第三步，生成摘要。

### 初始化缓冲区

MD5固定初始化四个值作为初始化。之后的生成摘要的计算，都是依据输入的消息，对这四个值做各种变换。

```c
A = 0x67452301;
B = 0xEFCDAB89;
C = 0x98BADCFE;
D = 0X10325476;
```

### 消息填充

原始数据长度加上填充长度与附加信息需要满足对512求余为0，也就是整个消息能被划分为若干512位的块，其中附加信息用来表示原始数据的长度，固定为64位，附加在填充后的数据末尾；填充位则首先固定需要填充1，然后填充0直到满足总长度对512求余为0的数学关系。

### 生成摘要

生成摘要的过程就很麻烦了，简单点讲就是用那几个512的块，对ABCD四个初始化向量做加法、乘法、移位、求余等一系列运算，这里是逆向专题，不需要了解细节。

## 完整实现

以下给出C伪代码，参考一下即可：

```c
// ================================================================
// 第一段：初始化代码 (结构体、常量表、上下文初始化)
// ================================================================

#include <stdint.h>
#include <string.h>

// 1. 循环左移宏
#define ROTATE_LEFT(x, n) (((x) << (n)) | ((x) >> (32 - (n))))

// 2. 四轮非线性函数
#define F(x, y, z) (((x) & (y)) | ((~x) & (z)))
#define G(x, y, z) (((x) & (z)) | ((y) & (~z)))
#define H(x, y, z) ((x) ^ (y) ^ (z))
#define I(x, y, z) ((y) ^ ((x) | (~z)))

// 3. 64步的常量表 T[i] = abs(sin(i+1)) * 2^32 (十六进制)
static const uint32_t K[64] = {
    0xd76aa478, 0xe8c7b756, 0x242070db, 0xc1bdceee,
    0xf57c0faf, 0x4787c62a, 0xa8304613, 0xfd469501,
    0x698098d8, 0x8b44f7af, 0xffff5bb1, 0x895cd7be,
    0x6b901122, 0xfd987193, 0xa679438e, 0x49b40821,
    0xf61e2562, 0xc040b340, 0x265e5a51, 0xe9b6c7aa,
    0xd62f105d, 0x02441453, 0xd8a1e681, 0xe7d3fbc8,
    0x21e1cde6, 0xc33707d6, 0xf4d50d87, 0x455a14ed,
    0xa9e3e905, 0xfcefa3f8, 0x676f02d9, 0x8d2a4c8a,
    0xfffa3942, 0x8771f681, 0x6d9d6122, 0xfde5380c,
    0xa4beea44, 0x4bdecfa9, 0xf6bb4b60, 0xbebfbc70,
    0x289b7ec6, 0xeaa127fa, 0xd4ef3085, 0x04881d05,
    0xd9d4d039, 0xe6db99e5, 0x1fa27cf8, 0xc4ac5665,
    0xf4292244, 0x432aff97, 0xab9423a7, 0xfc93a039,
    0x655b59c3, 0x8f0ccc92, 0xffeff47d, 0x85845dd1,
    0x6fa87e4f, 0xfe2ce6e0, 0xa3014314, 0x4e0811a1,
    0xf7537e82, 0xbd3af235, 0x2ad7d2bb, 0xeb86d391
};

// 4. 64步的循环左移位数
static const uint32_t S[64] = {
    7, 12, 17, 22, 7, 12, 17, 22, 7, 12, 17, 22, 7, 12, 17, 22,
    5,  9, 14, 20, 5,  9, 14, 20, 5,  9, 14, 20, 5,  9, 14, 20,
    4, 11, 16, 23, 4, 11, 16, 23, 4, 11, 16, 23, 4, 11, 16, 23,
    6, 10, 15, 21, 6, 10, 15, 21, 6, 10, 15, 21, 6, 10, 15, 21
};

// 5. MD5上下文结构体
typedef struct {
    uint32_t state[4];   // 四个寄存器 (A, B, C, D)
    uint64_t count;      // 已处理的字节数 (最大 2^64 位)
    uint8_t  buffer[64]; // 512位的输入缓存块
} MD5_CTX;

// 6. 上下文初始化函数 (设置幻数)
void md5_init(MD5_CTX *ctx) {
    ctx->state[0] = 0x67452301; // A
    ctx->state[1] = 0xefcdab89; // B
    ctx->state[2] = 0x98badcfe; // C
    ctx->state[3] = 0x10325476; // D
    ctx->count = 0;
    memset(ctx->buffer, 0, sizeof(ctx->buffer));
}

// ================================================================
// 第二段：填充代码 (添加 0x80、补零、附加原始长度)
// ================================================================

// 注：此处假设外部调用者已通过 md5_update 将数据送入 buffer，
// 此函数负责处理最后不足512位的尾部数据并收尾。
void md5_finalize(MD5_CTX *ctx, uint8_t digest[16]) {
    uint64_t bit_len = ctx->count * 8; // 原始数据的总位数
    uint32_t used = ctx->count % 64;   // 当前缓存块已占用的字节数

    // ----- 步骤1：追加一个 '1' 位 (0x80) -----
    ctx->buffer[used] = 0x80;

    // ----- 步骤2：追加若干个 '0' 位 -----
    if (used < 56) {
        // 清空从 used+1 到 55 的字节，使当前块留出最后8字节存放长度
        memset(ctx->buffer + used + 1, 0, 56 - used - 1);
    } else {
        // 如果已用字节 >= 56，当前块放不下长度，需清空当前块剩余部分并处理
        memset(ctx->buffer + used + 1, 0, 64 - used - 1);
        md5_transform(ctx->state, ctx->buffer); // 先处理当前已填满的块
        // 清空下一块的前56字节，为存放长度做准备
        memset(ctx->buffer, 0, 56);
    }

    // ----- 步骤3：附加64位原始长度 (小端序) -----
    for (int i = 0; i < 8; i++) {
        ctx->buffer[56 + i] = (uint8_t)(bit_len >> (i * 8));
    }

    // 处理包含填充和长度的最后一块
    md5_transform(ctx->state, ctx->buffer);

    // ----- 步骤4：输出128位摘要 (小端序) -----
    for (int i = 0; i < 4; i++) {
        for (int j = 0; j < 4; j++) {
            digest[i * 4 + j] = (uint8_t)(ctx->state[i] >> (j * 8));
        }
    }
}

// ================================================================
// 第三段：处理代码 (512位分块压缩函数，包含四轮64步核心运算)
// ================================================================

void md5_transform(uint32_t state[4], const uint8_t block[64]) {
    uint32_t a = state[0], b = state[1], c = state[2], d = state[3];
    uint32_t X[16]; // 存放当前块的16个32位子分组
    int i;

    // ----- 1. 将64字节块转换为16个32位小端序整数 -----
    for (i = 0; i < 16; i++) {
        X[i] = (uint32_t)block[i * 4] |
               ((uint32_t)block[i * 4 + 1] << 8) |
               ((uint32_t)block[i * 4 + 2] << 16) |
               ((uint32_t)block[i * 4 + 3] << 24);
    }

    // ----- 2. 主循环：四轮，共64步 -----
    for (i = 0; i < 64; i++) {
        uint32_t f, g; // 非线性函数输出 和 子分组索引

        // 根据步骤选择对应的非线函数与索引
        if (i < 16) {          // 第一轮
            f = F(b, c, d);
            g = i;
        } else if (i < 32) {   // 第二轮
            f = G(b, c, d);
            g = (5 * i + 1) % 16;
        } else if (i < 48) {   // 第三轮
            f = H(b, c, d);
            g = (3 * i + 5) % 16;
        } else {               // 第四轮
            f = I(b, c, d);
            g = (7 * i) % 16;
        }

        // 核心迭代运算 (等效于寄存器循环右移)
        uint32_t temp = d;
        d = c;
        c = b;
        b = b + ROTATE_LEFT((a + f + K[i] + X[g]), S[i]);
        a = temp;
    }

    // ----- 3. 将本块计算结果累加到状态中 -----
    state[0] += a;
    state[1] += b;
    state[2] += c;
    state[3] += d;
}
```



# SHA原理

SHA算法目前主要用的是SHA-2，这个按长度分包含了SHA-224、SHA-256、SHA-384、SHA-512。SHA-1是更早年的版本，现在也不用了，SHA-3则是最新的版本，但是由于SHA-2太强了所以现在很少见。作为一个系列其实现，其实步骤都是差不多的，哈希算法的实现无非都是初始化向量、填充、计算摘要，MD5已经完整讲过了这里就不赘述了，以SHA-256为例子，其特征主要是用了八个初始化向量。

```c
H0 = 0x6a09e667;
H1 = 0xbb67ae85;
H2 = 0x3c6ef372;
H3 = 0xa54ff53a;
H4 = 0x510e527f;
H5 = 0x9b05688c;
H6 = 0x1f83d9ab;
H7 = 0x5be0cd19;
```

# 逆向分析

哈希算法的特征，就算不看代码也很明显了，因为任何哈希算法是固定长度输出的，看出是哈希算法以后，只要简单结合初始化向量，就能分析出是什么算法，因此这类算法在CTF逆向题中占比不高，简单看看就行。

随便找一题，一开始就能看到MD5的四个向量，算法也就基本确认了，不过注意IDA不一定总是以十六进制显示的，记得自己转换一下。
