# 原理

Off by one指的就是只溢出了一字节，栈和堆都有这个概念，不过似乎堆的更多一点。堆中的溢出off by one，通常用来覆盖掉chunk与chunk之间的结构，使chunk重叠、以此泄漏其它块数据或覆盖其它块数据；如果溢出的一个字节为Null字节，又叫Off by null，这时候可以使size的低位prev_inuse被清空，则前一块便会被认为是free的，就可以结合其它打法去做题了。

glibc 2.28前，prev_inuse被清空，会导致prev_size启用，可以通过伪造prev_size造成块重叠，而2,28后则会检查前一块的size和当前块的prev_size是否相同

```C
/* consolidate backward */ 
if (!prev_inuse(p)) {
	prevsize = prev_size (p);
	size += prevsize;
	p = chunk_at_offset(p, -((long) prevsize));
	/* 2.28以后加入了以下代码 */
	if (__glibc_unlikely (chunksize(p) != prevsize)) 
		malloc_printerr ("corrupted size vs. prev_size while consolidating");
		unlink_chunk (av, p); 
}
```
